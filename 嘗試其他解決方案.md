# Visual Studio 2026 編譯問題 - 其他解決方案

## 當前問題

`node-gyp` 無法識別 Visual Studio 2026（版本 18），導致編譯失敗。

## 解決方案

### 方案 1：使用 npm config 設置（在編譯前運行）

```cmd
npm config set msvs_version 2022
npm rebuild @tensorflow/tfjs-node --build-from-source
```

### 方案 2：手動設置環境變數後編譯

```cmd
set GYP_MSVS_VERSION=2022
set npm_config_msvs_version=2022
set VisualStudioVersion=17.0
npm rebuild @tensorflow/tfjs-node --build-from-source
```

### 方案 3：直接使用 node-gyp 編譯

```cmd
cd node_modules\@tensorflow\tfjs-node
node-gyp configure --msvs_version=2022
node-gyp build
```

### 方案 4：檢查 Visual Studio 安裝

確保 Visual Studio 2026 已安裝以下組件：
- Desktop development with C++ workload
- Windows SDK
- C++ CMake tools

### 方案 5：考慮降級到 Visual Studio 2022

如果以上方案都不行，可以考慮安裝 Visual Studio 2022，因為 `node-gyp` 對它的支持更好。
